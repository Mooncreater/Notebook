# 02 · Base64 — 伪装，不是加密

---

## 一、这是什么？

**Base64 不是加密，是换马甲。**

它把**任何数据**（文字、图片、声音）转成 **64 个安全字符**的组合。

> 为什么 64 个？因为 2^6 = 64，刚好用 6 个二进制位表示一个字符。

---

## 二、为什么需要它？—— 三个真实场景

### 场景 1：邮件附件
邮件协议最初只支持纯文本。你要发一张图片怎么办？
→ 把图片的二进制数据用 Base64 转成文本，塞进邮件正文。

### 场景 2：网页内嵌图片
```html
<img src="data:image/png;base64,iVBORw0KGgo...==" />
```
不用额外发 HTTP 请求，图片直接"写"在 HTML 里。

### 场景 3：URL 安全传输
二进制数据不能直接放 URL 里（URL 只允许某些字符）。
→ Base64 编码后，全是安全字符，随便传。

---

## 三、原理 —— 一步步拆解

### Base64 的 64 个字符表

```
A-Z (0-25)
a-z (26-51)
0-9 (52-61)
+   (62)
/   (63)
=   (填充符号)
```

### 核心规则

```
原始数据 = 二进制 8位一组（字节）
Base64 = 二进制 6位一组（然后查表转字符）
```

### 手把手编码 "Man"

#### Step 1：每个字母转 ASCII 码

```
M → 77  (二进制：01001101)
a → 97  (二进制：01100001)
n → 110 (二进制：01101110)
```

#### Step 2：把三个字节的二进制串起来

```
01001101 01100001 01101110
```

#### Step 3：重新按 6 位一组切

```
010011  010110  000101  101110
```

#### Step 4：每组转十进制，查 Base64 表

| 6位二进制 | 十进制 | 查表 |
|----------|--------|------|
| 010011 | 19 | **T** |
| 010110 | 22 | **W** |
| 000101 | 5  | **F** |
| 101110 | 46 | **u** |

#### Step 5：结果

```
Man → TWFu
```

### 位数不对怎么办？—— 填充 `=`

如果数据不是 3 的倍数：

- 缺 1 个字节 → 补 2 个 `=`
- 缺 2 个字节 → 补 1 个 `=`

```
M      → 77 → 01001101 → 010011 010000(补零) → T Q = =
"Ma"   → M(77) a(97) → 01001101 01100001 → 010011 010110 000100(补零) → T W E =
"Man"  → 正好3字节 → T W F u （无等号）
```

## 四、手把手：你在 PowerShell 里对 "Hello" 编码

### 编码

```powershell
[System.Convert]::ToBase64String([System.Text.Encoding]::UTF8.GetBytes("Hello"))
```

输出：
```
SGVsbG8=
```

### 解码

```powershell
[System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String("SGVsbG8="))
```

输出：
```
Hello
```

## 五、动手实验：图片转 Base64

### 1. 把一张图片转成 Base64 文本

```powershell
$bytes = [System.IO.File]::ReadAllBytes("D:\test.png")
$b64 = [System.Convert]::ToBase64String($bytes)
$b64 | Out-File "D:\test_base64.txt"
```

打开 `D:\test_base64.txt`，看到一大串字符，这就是图片的马甲。

### 2. 从 Base64 还原图片

```powershell
$b64 = Get-Content "D:\test_base64.txt"
$bytes = [System.Convert]::FromBase64String($b64)
[System.IO.File]::WriteAllBytes("D:\test_restored.png", $bytes)
```

## 六、Base64 能用来"加密"吗？

### 很多人以为这样安全

```powershell
# "加密"
$password = "123456"
$encoded = [System.Convert]::ToBase64String([System.Text.Encoding]::UTF8.GetBytes($password))
Write-Host "加密后的密码：$encoded"
# 输出：MTIzNDU2
```

### 但"解密"不用密钥

```powershell
$decoded = [System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String("MTIzNDU2"))
Write-Host "解密：$decoded"
# 输出：123456
```

**Base64 不是加密，是编码。** 算法公开，没有密钥，任何人拿到都能解。

> 类比：Base64 = 把汉字写成拼音，不是把内容藏起来，只是换了一种写法。

---

## 七、Base64 变种

| 变种 | 区别 | 用途 |
|------|------|------|
| 标准 Base64 | 用 `+` `/` `=` | 邮件、通用 |
| URL-Safe Base64 | `+` 变 `-`，`/` 变 `_`，去 `=` | URL 参数、JWT Token |
| Base32 | 32个字符 | 人眼友好（不含大小写混淆） |
| Base16（Hex） | 16个字符（0-9 A-F） | 颜色值 #FF00FF、哈希展示 |

## 八、练习题

### 练习 1
解码 `SGVsbG8gV29ybGQh`

<details><summary>答案</summary>

```powershell
[System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String("SGVsbG8gV29ybGQh"))
```
输出：`Hello World!`

</details>

### 练习 2
编码 `Codex`

<details><summary>答案</summary>

```powershell
[System.Convert]::ToBase64String([System.Text.Encoding]::UTF8.GetBytes("Codex"))
```
输出：`Q29kZXg=`

注意末尾的 `=`——"Codex" 5 个字节，5 % 3 = 2，缺 1 个字节 → 补 1 个 `=`。

</details>

### 练习 3
下面这串是 Base64，解码它（提示：是中文的 Base64）：

`5L2g5aW9` → 解码后是什么？

<details><summary>答案</summary>

```powershell
[System.Text.Encoding]::UTF8.GetString([System.Convert]::FromBase64String("5L2g5aW9"))
```
输出：`你好`

</details>

## 🎯 本章总结

| 概念 | 一句话 |
|------|--------|
| Base64 | 把二进制数据换成 64 个安全字符，方便在文本系统里传输 |
| 不是加密 | 算法公开，没密钥，任何人拿到都能还原 |
| 填充 `=` | 位数不够3的倍数时补零填位 |
| 用途 | 邮件附件、网页内嵌图片、JWT Token、API 认证头 |
