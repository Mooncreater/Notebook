# AI 加速器（DL Accelerator）

!!! tip "核心要点"
    为什么需要 AI 加速器？CPU/GPU 处理神经网络**能效比低**。AI 加速器通过**专用数据通路**和**领域专用架构（DSA）**实现高能效。核心分析维度：计算特性 + 访存特性。

---

## 1. 为什么需要 AI 加速器？

| 平台 | 类比 | 特点 |
|------|------|------|
| CPU | 一个大学生 | 通用性强，但算力/能效低 |
| GPU | 100 个小学生 | 并行度高，但仍不够专用 |
| DL Accelerator | 一个偏科生 | **极致能效**，只做深度学习 |

**关键数据**：
- Google Brain：1.6 万个 CPU 核跑数天完成猫脸识别
- AlphaGo vs 李世石：1202 CPU + 176 GPU

## 2. 深度学习算法分析

### 两大特性

| 特性 | 关注点 |
|------|--------|
| **计算特性** | 是否存在固定的、重复的计算模式？ |
| **访存特性** | 数据访问的局部性、数据与计算的关系 |

### 典型算子分析

| 算子 | 计算特性 | 访存特性 |
|------|----------|----------|
| **卷积（Conv）** | 矩阵乘矩阵/向量 | Burst + Stride |
| **激活函数** | 向量运算 | Burst |
| **池化（Pooling）** | 窗口比较/平均 | Burst + Stride |
| **全连接（FC）** | 矩阵乘向量 | Burst |

**Burst**：突发传输（连续地址）
**Stride**：跳着访问（固定间隔）

---

## 3. 加速器设计思路

### 设计哲学

1. **专用数据通路**：针对 Conv/MatMul 等核心算子的专用硬件
2. **减少数据搬运**：片上存储层次，数据靠近计算单元
3. **固定计算模式**：脉动阵列（Systolic Array）用于矩阵乘

### 通用 CPU 功耗分析

大部分功耗花在**控制逻辑**（分支预测、乱序调度、寄存器重命名...），而非实际计算。

:arrow_right: AI 加速器**去掉不必要的控制逻辑**，把面积和功耗给计算单元。

---

## 4. 脉动阵列（Systolic Array）

### 原理

传统 CPU：Memory → PE → Memory（每次计算都要访存）

脉动阵列：Memory → PE1 → PE2 → PE3 → Memory
- 数据像心跳一样在 PE 间脉动传播
- **一次加载，多次使用**（数据复用）

### 结构（2D Systolic Array）

`
     ┌───┐ ┌───┐ ┌───┐
  →  │PE │→│PE │→│PE │  →
     └───┘ └───┘ └───┘
     ┌───┐ ┌───┐ ┌───┐
  →  │PE │→│PE │→│PE │  →
     └───┘ └───┘ └───┘
     ┌───┐ ┌───┐ ┌───┐
  →  │PE │→│PE │→│PE │  →
     └───┘ └───┘ └───┘
`

每个 PE：Right = Left; Down = Upper; Cell += Upper × Left

### 应用

- **TPU v1**：256×256 MAC 脉动阵列，占芯片 24% 面积
- 优势：极高的数据复用率，低功耗
