# 消影	
	
消影（Hidden Surface Removal）是确定场景中哪些面可见、哪些面被遮挡的过程，是渲染管线中的关键环节。	
	
## 1. Warnock 算法	
	
Warnock 算法由 John Warnock 于 1969 年提出，基于**分治策略**（Divide & Conquer），通过递归细分屏幕区域来判断可见面。	
	
### 递归流程	
	
对当前屏幕区域，分析多边形与其关系：	
	
1. 所有多边形与该区域无交集 :arrow_right: 填背景色，返回	
2. 只有一个多边形覆盖该区域 :arrow_right: 直接填充，返回	
3. 若区域已缩小到单个像素 :arrow_right: 取深度最近的填充，返回	
4. 否则 :arrow_right: **将区域四等分**，递归处理每个子区域	
	
### 特点	
	
- 递归深度不确定，可能一直细分到像素级	
- 天然适合层次化场景	
	
## 2. Z-Buffer 算法	
	
**Z-Buffer**（深度缓冲）是当前 GPU 最常用的消影算法。为每个像素存储一个深度值，通过比较深度确定可见性。	
	
### 数据结构	
	
- **帧缓存（Frame Buffer）**：存储每个像素的颜色值	
- **深度缓存（Z-Buffer）**：存储每个像素的深度值，初始化为 $\infty$	
	
### 算法流程	
	
```	
对所有多边形：	
  对该多边形覆盖的每个像素 (x, y)：	
    计算深度 z = depth(x, y)	
    if z < zbuffer[x][y]:	
        zbuffer[x][y] = z	
        framebuffer[x][y] = color(x, y)	
```	
	
### 增量深度计算	
	
利用平面方程 $Ax + By + Cz + D = 0$，从 $(x, y)$ 的深度可递推 $(x+1, y)$：	
	
$$z(x+1, y) = z(x, y) - \frac{A}{C}$$	
	
### 特点	
	
- ✅ 实现简单，硬件友好（当今 GPU 内置）	
- ✅ 不需要对多边形排序	
- ✅ 增量计算深度，效率高	
- ❌ 需要额外内存存储深度缓冲	
- ❌ 无法直接处理透明物体	
	
## 3. Painter's Algorithm 画家算法	
	
**思路**：从远到近依次绘制，后画的覆盖前面的。	
	
- 对多边形按深度从远到近排序	
- 按排序顺序依次绘制	
	
⚠️ 问题：可能存在**循环重叠**（A 遮挡 B，B 遮挡 C，C 遮挡 A）	
	
:arrow_right: 解决方法：将多边形切割，直到消除循环重叠	
	
### BSP 树（Binary Space Partitioning Tree）	
	
**二叉空间分割树**：预先将场景组织为二叉树	
	
1. 选择一个多边形所在平面作为分割平面	
2. 将空间划分为前后两个子空间	
3. 递归对每个子空间进行分割	
4. 直到所有多边形归入叶节点	
	
**绘制**：根据视点与根节点的前后关系，递归遍历 BSP 树（从远到近绘制）	
	
:arrow_right: 🌲 可预先构造，渲染时只需 $O(n)$ 遍历，适合静态场景	
	
## 4. Ray Casting 算法	
	
**射线投射**：从视点穿过每个像素发射射线，取最近交点着色。	
	
### 流程	
	
1. 对每个像素，生成一条从视点发射的射线 $\mathbf{P}(t) = \mathbf{O} + t\mathbf{D}$	
2. 计算射线与场景中所有物体的交点	
3. 取 $t$ 最小（最近）的交点	
4. 在该交点处计算光照，填入像素颜色	
	
### 特点	
	
- ✅ 天然处理消影（最近交点即最可见面）	
- ✅ 可扩展为完整光线追踪（反射、折射）	
- ❌ 求交计算量大，需加速结构（BVH / KD-Tree）

### 消影算法对比	
	
| 算法 | 排序需求 | 硬件友好 | 透明度 | 适用场景 |	
|------|----------|----------|--------|----------|	
| Warnock | 不需要 | 差 | 间接 | 历史意义 |	
| Z-Buffer | 不需要 | GPU 内置 | 需单独处理 | 实时渲染 |	
| Painter's | 需要 | 一般 | 天然支持 | 2.5D / 早期 |	
| Ray Casting | 不需要 | RT Core | 天然支持 | 离线渲染 |
