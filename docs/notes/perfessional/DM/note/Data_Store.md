# 数据仓库与 OLAP	
	
## OLTP vs OLAP	
	
| | OLTP | OLAP |	
|--|------|------|	
| 用途 | 日常事务处理 | 数据分析与决策 |	
| 操作 | 增删改查（短事务） | 复杂查询（聚合分析） |	
| 数据 | 当前、细节、关系型 | 历史、汇总、多维 |	
| 用户 | 一线操作人员 | 管理层、分析师 |	
| DB 大小 | GB | TB ~ PB |	
	
## 数据仓库特点	
	
- **面向主题**：按业务主题组织数据	
- **集成性**：整合多个异构数据源	
- **时变性**：存储历史数据，带时间标签	
- **非易失性**：只读访问，不进行更新操作	
	
## ETL 流程	
	
数据从源系统到数据仓库的三步：	
	
1. **Extract（抽取）**：从多个源系统提取数据	
2. **Transform（转换）**：数据清洗、格式统一、去重	
3. **Load（加载）**：加载到数据仓库中	
	
## 多维数据模型	
	
### 星型模型（Star Schema）	
	
中心一张**事实表**，周围多张**维度表**：	
- 事实表：存储度量值（销售额、数量）和外键	
- 维度表：存储描述性属性（时间、地点、产品）	
	
### 雪花模型（Snowflake Schema）	
	
星型模型的变体，维度表进一步规范化，形成层次结构。	
	
### 事实星座（Fact Constellation）	
	
多个事实表共享维度表，用于复杂业务场景。	
	
## 数据立方体操作	
	
- **Roll-up（上卷）**：沿维度层次向上聚合（城市 → 省份）	
- **Drill-down（下钻）**：沿维度层次向下细化（年 → 月 → 日）	
- **Slice（切片）**：固定一个维度值，看其余维度（只看 2025 年）	
- **Dice（切块）**：固定多个维度范围（2024-2025 年 + 华东区）	
- **Pivot（旋转）**：交换坐标轴视角	
	
## 数据立方体的物化	
	
物化（Materialization）：预计算并存储某些方体的汇总结果。	
	
- **完全物化**：所有组合都计算，空间爆炸	
- **部分物化**：只计算常用组合，折中方案	
- **不物化**：全部实时计算，查询慢	
	
:arrow_right: 冰山方体（Iceberg Cube）：只物化超过阈值的组合，节省空间。
