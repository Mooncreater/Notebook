# Git 规范指南

本文档旨在建立统一的 Git 使用规范，帮助团队更高效地协作，保持代码历史的清晰和可追溯性。

## 📋 规范列表

### [提交规范](commit.md)
统一的提交信息格式，让代码历史更加清晰易懂。采用 Conventional Commits 规范，支持自动生成 CHANGELOG。

### [分支规范](branch.md)
标准化的分支命名和管理策略，确保多人协作时的代码管理井然有序。

### [工作流规范](workflow.md)
定义完整的 Git 工作流程，包括功能开发、Bug 修复、版本发布等场景的操作指南。

## 🎯 为什么要统一规范？

| 规范类型 | 解决的问题 |
|:---|:---|
| **提交规范** | 提交信息混乱，无法快速定位修改内容 |
| **分支规范** | 分支命名随意，难以区分功能、修复、发布等 |
| **工作流规范** | 协作流程不清晰，容易产生代码冲突 |

## 🚀 快速开始

如果你是第一次接触这些规范，建议按以下顺序阅读：

1. 先了解 **[提交规范](commit.md)** - 这是最基础也最常用的规范
2. 然后学习 **[分支规范](branch.md)** - 了解如何管理不同的分支
3. 最后熟悉 **[工作流规范](workflow.md)** - 掌握完整的协作流程
