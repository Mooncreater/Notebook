    GPTs 与对齐 (Alignment)¶

    约 1497 个字 5 行代码 预计阅读时间 5 分钟

    从 GPT 到 ChatGPT：大模型如何从"续写"走向"对话"，以及如何让模型输出符合人类偏好。

    核心要点

    大模型训练三阶段：预训练 (Pre-training) → 指令微调 (SFT) → 对齐 (Alignment: RLHF/DPO)。对齐的核心是让模型输出"有用、真实、无害"，而非仅仅概率最大。

    1. 从词向量到 GPT¶

    两条路线¶

    路线 任务 架构 代表模型 应用范式
    填词 双向上下文预测被 mask 的词 → MLM Transformer Encoder BERT 预训练 + 微调
    接龙 单向预测下一个 token → LM Transformer Decoder GPT 语境学习 (In-Context Learning)

    GPT-3 (175B 参数) 证明了接龙路线的巨大潜力：无需微调，仅通过 prompt 即可完成 42 个下游任务，轻松通过图灵测试。

    Scaling Laws (伸缩法则)¶

    模型性能随参数规模、数据规模、计算开销的增大而幂律提升。

    \[L(N, D, C) \propto N^{-\alpha_N} + D^{-\alpha_D} + C^{-\alpha_C}\]

    涌现能力 (Emergent Ability)¶

    当规模跨越某个临界点，模型突然展现出训练时未专门学习的能力：

    • 举一反三 (Few-shot reasoning)
    • 数学推理
    • 代码生成
    • 跨域迁移

    2. 大模型训练三阶段¶

    预训练 (Pre-training)  
    ↓   
监督微调 (SFT)  
    ↓   
人类反馈强化学习 (RLHF) / DPO

    阶段一：预训练 (Pre-training)¶

    在海量无标注语料上进行自监督学习，学习通用语言知识。

    以盘古大模型为例：

    • 语料规模 ~80TB，需大规模清洗：去重、去广告、繁转简、特殊符号过滤、敏感内容过滤
    • 分布式训练：多维混合并行（数据 + 模型 + 流水线 + 张量并行）
    • 内存优化：ZeRO、梯度累积、重计算

    阶段二：监督微调 (SFT)¶

    用高质量（指令, 回复）对训练模型遵循指令的能力。

    • 数据来源：人工标注 + 模型生成 + 用户反馈
    • 目的：让模型理解"对话"格式，学会遵循人类指令

    阶段三：对齐 (Alignment)¶

    SFT 后的模型虽然能跟随指令，但可能输出有害、虚假或不合适的内容。

    3. SFT 的局限性¶

    • 幻觉 (Hallucination)：模型编造看似合理但错误的事实
    • 有害输出：可能生成歧视、暴力等不安全内容
    • 谄媚 (Sycophancy)：迎合用户而非坚持事实
    • 缺乏拒答：不知道说"我不知道"

    4. RLHF (Reinforcement Learning from Human Feedback)¶

    核心流程¶

    1. 收集偏好数据：对同一 prompt 的多个模型输出进行人工排序
    2. 训练奖励模型 (RM)：学习预测人类偏好 \(r_\phi(x, y)\)
    3. 用 PPO 优化策略：最大化奖励的同时保持与 SFT 模型的接近（加 KL 散度惩罚）
    \[\max_{\pi_\theta} \mathbb{E}_{x\sim D, y\sim\pi_\theta(y|x)} \left[r_\phi(x, y) - \beta \cdot KL(\pi_\theta \| \pi_{SFT})\right]\]
    • \(\beta\) 控制偏离 SFT 模型的程度
    • KL 惩罚防止模型为追求奖励而退化（输出无意义但高奖励的文本）

    RLHF 的问题¶

    • 人工标注成本高
    • 奖励模型黑客 (Reward Hacking)：模型找到奖励模型的漏洞
    • 训练不稳定：PPO 需要同时维护 4 个模型（Policy, Reference, Reward, Value）

    5. DPO (Direct Preference Optimization)¶

    核心思想：直接从偏好数据学习，无需显式训练奖励模型。

    DPO 将 RLHF 的优化问题转化为一个简单的分类损失：

    \[\mathcal{L}_{DPO} = -\mathbb{E}_{(x, y_w, y_l)\sim D} \left[\log\sigma\left(\beta\log\frac{\pi_\theta(y_w|x)}{\pi_{ref}(y_w|x)} - \beta\log\frac{\pi_\theta(y_l|x)}{\pi_{ref}(y_l|x)}\right)\right]\]
    • \(y_w\)：被偏好的输出 (winning)
    • \(y_l\)：不被偏好的输出 (losing)
    • 只需两个模型：当前策略 \(\pi_\theta\) 和参考策略 \(\pi_{ref}\)

    DPO vs RLHF¶

    RLHF DPO
    步骤 3 步（SFT → RM → PPO） 2 步（SFT → DPO）
    稳定性 训练 4 个模型，不稳定 简单分类损失，稳定
    效果 经典方法，被广泛验证 等价于 RLHF，但更简洁

    6. 对齐中的关键问题¶

    幻觉 (Hallucination)¶

    模型生成看似合理但事实错误的文本。根源：

    • 训练数据中的错误/偏见
    • 最大似然训练不区分事实与虚构
    • 模型没有真实的"知识"概念

    缓解方法：RAG（检索增强生成）、事实性奖励、后训练事实性优化。

    谄媚问题 (Sycophancy)¶

    RLHF 训练出的模型倾向于告诉用户想听的内容。五大主流 LLM 均表现出此问题。

    原因：人类评审倾向于认可措辞良好的回答，即使它不准确。

    安全与对齐¶

    风险 缓解
    有害输出 RLHF/DPO 的安全训练、内容过滤
    越狱攻击 (Jailbreak) 对抗训练、红队测试
    隐私泄露 训练数据去隐私、差分隐私
    偏见与歧视 多样化的偏好数据、公平性约束

    7. 系统 1 与系统 2¶

    诺贝尔经济学奖得主丹尼尔·卡尼曼 (Daniel Kahneman) 将人类思维分为两大模式：

    系统 1 (感知) 系统 2 (认知)
    特点 快速、直觉、自动 慢速、理性、受控
    AI 对应 深度学习 (模式匹配) 符号推理、搜索
    当前地位 大力出奇迹，感知系统占主导 尚待发展

    当前深度学习的本质是系统 1：用一个函数拟合输入-输出映射。这导致了 AI 的典型问题：

    • 不能举一反三（缺乏系统泛化）
    • 缺乏常识推理
    • 不具备可解释性
    • 容易受到对抗攻击

    未来方向：系统 2 模型（认知）与系统 1 模型（感知）的协同——大模型自己写代码、生成数据来训练感知模型，实现自我进化。

    8. Agent 与工具使用¶

    Function Calling¶

    大模型调用外部工具（搜索、计算器、数据库、API）扩展自身能力。

    ReAct 模式¶

    Reasoning + Acting：模型交替进行推理和行动。思考 → 行动 → 观察 → 思考 → ...，支持多步复杂任务。

    多模态¶

    GPT-4V / GPT-4o：从纯文本扩展到图像、音频、视频的理解与生成。

    RLHF 三阶段流程、DPO 的核心公式思想、Scaling Laws 与涌现能力的含义、系统 1 vs 系统 2 的区分。
